Aircraft Tracker

Overview: Developed for the Circuit Communications class, this project utilizes a USB RTL-SDR dongle connected to an antenna to capture live radio frequency signals from nearby aircraft.

How it Works: The system intercepts Automatic Dependent Surveillance–Broadcast (ADS-B) signals. By processing these raw signals, the system extracts critical data such as the aircraft's type and unique callsign. This project focused purely on hardware configuration and signal processing theory without high-level programming languages.

Self Balancing Robot

Overview: Created for the Embedded Systems class, this robot repurposes a 3-wheel chassis into a 2-wheel self-balancing vehicle controlled by a Texas Instruments TIVA LaunchPad.

Technical Details: The system relies on a feedback loop where motors adjust their speed and direction based on real-time data. An oscilloscope was utilized during development to visualize signals and tune the stability, ensuring the robot could stand upright autonomously.

Action Shot

Water Monitoring System

Overview: A Sensor Applications class project designed to read and visualize water quality data remotely.

Pipeline: Raw data is gathered by sensors and processed by an Arduino. The Arduino translates this data and sends it to LabVIEW for local display. LabVIEW then pushes the data to the ThingSpeak API, which serves as the backend for a live website hosted on GitHub, allowing for real-time remote monitoring.

Dashboard View

FPGA Reaction Timer

Overview: A Microprocessor Architecture project built on an Intel FPGA using Verilog, simulating the "start lights" found in racing.

Functionality: The board's LEDs simulate a countdown sequence. Once the light turns green, an internal millisecond timer begins. The user stops the timer via a physical button, and the precise reaction time is displayed on the board's 7-segment display modules.

Device in Action

Line Following Robot

Overview: Developed for Circuits 2, this autonomous robot navigates a track by detecting contrast between the floor and a line.

Logic: Controlled by an Arduino, the system uses two infrared (IR) sensors mounted at the front. The control logic constantly adjusts motor power to keep the line centered between the two sensors, correcting course immediately if a sensor detects the line.
